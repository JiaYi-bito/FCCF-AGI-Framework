Functional Causal Cognition Framework (FCCF)
A Cognitive Architecture Hypothesis for Artificial General Intelligence (Enhanced Fusion Version)
Author: JiaYi (Independent Researcher)
Date: March 2026
Abstract
Current artificial intelligence (AI) development faces a fundamental bottleneck: even the most advanced large language models struggle to truly "understand" the information they process. Existing AI systems rely primarily on large-scale representation learning to complete tasks through statistical correlation, but exhibit significant limitations in causal understanding, generalization ability, and continuous learning.
This paper proposes a cognitive architecture hypothesis—the Functional Causal Cognition Framework (FCCF). The framework is grounded in a core observation:
The essence of human cognition is not "feature statistics" but "functional understanding".
Through four mechanisms—causal chain tracing, functional abstraction, core anchoring, and dynamic updating—FCCF constructs a cognitive system capable of continuous growth. Its core pathway is:
Experience → Causality → Function → Abstraction → Transfer
This framework attempts to provide an alternative implementation path for Artificial General Intelligence (AGI) that departs from the "scale stacking" paradigm.
1. Background: The Predicament of Current AI
In recent years, AI development has been highly dependent on the "Scaling Law", i.e., improving performance by increasing parameter scale, data volume, and computing power.
However, this path is encountering diminishing marginal returns:
Knowledge freezing: Models cannot learn new knowledge in real time
Hallucination problem: Lack of genuine understanding
Limited generalization ability: Struggles to adapt to new scenarios
Extremely high energy consumption: Far lower efficiency than the human brain
The fundamental reason lies in:
Current AI is a "correlation-based" system, not a "causality-and-function-based" system.
2. First Principles: How Humans Understand the World
2.1 A Key Observation
When humans recognize a "cup", it is not because of its pixel features, but because:
It can hold liquid and be used for drinking water.
In other words:
Humans recognize "function" rather than "appearance".
2.2 The Essence of Function
Function is not inherent to an object but is determined by its usage:
The same object can have different functions in different scenarios.
Thus, we define:
Function = F(Object, Scenario, Action, Intention)
2.3 Hierarchical Structure of Abstraction
Concrete Function → Core Function → Ultimate Purpose
For example:
Stir-frying, steaming, frying → Cooking → Survival needs
This abstraction ability enables humans to understand the world across scenarios.
2.4 Core Cognitive Pathway
Experience → Causality → Function → Abstraction → Transfer
This pathway is the core mechanism of human intelligence.
3. Functional Causal Cognition Framework (FCCF)
3.1 Core Definition
FCCF is a cognitive system based on causal modeling and functional abstraction, aiming to achieve cross-domain understanding and decision-making with limited experience.
The essence of intelligence is not remembering the world, but understanding how the world works.
3.2 Core Formula
Intelligence Level ≈ Experience × Abstraction Ability × Causal Modeling Ability × Memory Capacity
3.3 Four Core Mechanisms
Mechanism 1: Causal Chain Tracing
Input → Process → Output → Purpose
By tracing causal chains, we understand the essence of things.
Mechanism 2: Functional Abstraction
Extracting core functions from specific behaviors:
Stir-frying, steaming, frying → Cooking
Mechanism 3: Core Anchoring
Core functions remain stable amid functional evolution.
Mechanism 4: Dynamic Updating
The cognitive system continuously cycles through:
Observation → Analysis → Update → Re-observation
4. System-Level Architecture (Enhanced Section)
To make the framework engineering-implementable, it can be decomposed into the following modules:
Module 1: Perception Layer
Inputs external information and converts it into structured states.
Module 2: World Model
State + Action → Future State
Used to predict future outcomes and support decision-making.
Module 3: Causal Engine
Constructs a causal relationship network:
Nodes: Events or states
Edges: Causal relationships
Used to explain "why things happen".
Module 4: Functional Abstraction Layer (Core)
Function = Mapping(State, Action → Goal)
Essence: A function is a structure that achieves a goal.
Module 5: Memory System
Semantic memory: Concepts and abstractions
Episodic memory: Specific experiences
Module 6: Learning Policy
Determines:
What to learn
What to ignore
How to abstract
Module 7: Value System
Defines goals and priorities:
Value = Reward / Goal / Priority
5. Cognitive Cycle (System Operation Mechanism)
The complete process is as follows:
Input
Prediction (World Model)
Causal Analysis
Functional Abstraction
Decision-Making
Action
Feedback
Update (Memory and Model)
This cycle enables the system to achieve continuous growth.
6. Core Advantages

Strong generalization ability: Cross-scenario transfer
Continuous growth: Experience-driven evolution
High interpretability: Clear causal pathways

Goal → Behavior → Feedback → Optimization

Interact with the environment to achieve autonomous learning and adaptation.

Intelligence is not about memorizing data, but about modeling causal structures;
It is not about fitting features, but about understanding functions.
